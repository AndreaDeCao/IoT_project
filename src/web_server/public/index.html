IoT Speed Camera Project

Embedded Software for the Internet of Things

Project Overview

The project integrates hardware sensors, wired communication and web-based visualisation in an IoT context.
This project consists of a small remote-controlled car and a scaled-down speed camera system.
The car is controlled through gyroscope-based gestures and communicates wirelessly with the rest of the system.
When the car passes through the speed camera, its speed is measured and compared against a predefined limit (0.75m/s right now).
A status LED turns green if the vehicle is within the limit and red if the limit is exceeded.
The measured data is then sent to a web server in JSON format, where it is stored and displayed for later analysis.

System Architecture

Arduino Nano (Controller Unit)

  • Reads MPU6050 gyroscope gestures
  • Sends movement commands via nRF24L01

Arduino Uno R4 (Car Unit)

  • Receives commands via nRF24L01
  • Drives motors through L298N module
  • Activates LED

ESP32 (Speed Camera Unit)

  • Reads IR sensors for speed measurement
  • Calculates velocity and detects overspeed
  • Sends data to the web server in JSON format

Web Server

  • Receives speed measurements in JSON format
  • Determines whether the speed exceeds the limit
  • Displays the speed value along with a corresponding LED (red if the driver should receive a fine, green otherwise)

How It Works

  1. The user controls the car through hand gestures detected by the MPU6050.
  2. The Arduino Nano processes the gesture data and sends commands via nRF24L01.
  3. The Arduino Uno R4 WiFi receives the commands and drives the motors through the L298N module.
  4. The car passes through the speed camera system based on the ESP32 and proximity sensors.
  5. The measured speed is compared against a threshold (0.75 m/s).
  6. The result is shown through LEDs and sent to the web server for logging and visualization.

Technical Details

Hardware and Software Components

  • Arduino UNO R4 WiFi – car unit
  • nRF24L01+ PA LNA (2.4 GHz) – wireless communication between car and controller
  • L298N Motor Driver – motor power supply and polarity control
  • 4x 5V Motors – one for each wheel
  • AZDelivery AZ-Nano V3 – controller unit
  • ESP32 – speed camera (velox)
  • 2x VCNL4010 – proximity sensors for speed measurement
  • Render – hosting platform for the website

Data Structures

  • MPU6050 data structure (x, y, z spatial position)
  • Torque vectoring algorithm
  • Finite State Machine (FSM) for the speed camera

Algorithms

  • Torque vectoring for differential speed control
  • 3D position mapping using MPU6050 (x, y, z axes)
